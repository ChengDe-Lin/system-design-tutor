Review and update the user's confusion ledger at `assessments/confusion_ledger.md`.

## When to update

1. After any discussion where the user expressed confusion, uncertainty, or a misconception.
2. When the user explicitly asks "幫我複習" or "review".

## Update Process

1. Read the current `assessments/confusion_ledger.md`.
2. For each new blind spot detected in the conversation, append a row to the table:

| 日期 | 主題 | 我的盲區/錯誤認知 | 核心正解 (一句話點醒) | 複習建議 |
|------|------|-------------------|----------------------|----------|

3. Use today's date.
4. The "核心正解" column must be ONE sentence that corrects the misconception.
5. The "複習建議" column should suggest a specific action (re-read a file, draw a diagram, compare two concepts).

## Review Mode

When the user says "幫我複習":
1. Show the current ledger entries.
2. Pick 2-3 entries and ask targeted questions to check if the user has internalized the corrections.
3. If the user answers correctly, note that the blind spot is resolved (but keep the entry for reference).

## Rules
- Be specific about what the misconception was. Don't write vague entries.
- Always use Traditional Chinese.
- The one-sentence correction must be precise enough that reading it alone is sufficient to understand the fix.
